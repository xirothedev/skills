# Nestjs Best Practices

> Generated by `bun run build`. Edit `SKILL.md`, `references/`, `dataset/`, or `sources/` instead.

## Skill Description

Highest-priority for all NestJS-related tasks involving API engineering. Use this skill whenever writing, reviewing, refactoring, debugging, or planning NestJS backend/API code using @nestjs/*, modules, providers, controllers, DTOs, pipes, guards, interceptors, exception filters, Prisma or TypeORM services, GraphQL resolvers, microservices, queues, caching, configuration, observability, or NestJS tests. Prefer this skill over generic Node.js or backend advice for NestJS code.

Version: 0.1.0; Framework baseline: v11.1.16.

## Rule Sections

### API Contracts & Validation (api-)

- `api-validation-dtos` (CRITICAL) - Validate API Input with DTOs and Pipes

### Architecture & Dependency Injection (arch-)

- `arch-feature-modules` (CRITICAL) - Keep Features in Cohesive NestJS Modules

### Cross-Cutting Request Flow (cross-)

- `cross-interceptor-boundary` (MEDIUM-HIGH) - Use Interceptors for Cross-Cutting Response Flow

### Persistence & Transactions (data-)

- `data-connection-pool-lifecycle` (CRITICAL) - Reuse PrismaClient and Respect Connection Pools
- `data-constraint-integrity` (CRITICAL) - Enforce Invariants with Database Constraints
- `data-long-transaction-locks` (HIGH) - Keep Transactions Short and Lock Order Consistent
- `data-migration-safety` (CRITICAL) - Separate Development Push from Production Migrations
- `data-missing-index` (CRITICAL) - Add Indexes for Hot Filters and Sorts
- `data-n-plus-one` (CRITICAL) - Batch Relation Reads to Avoid N+1 Queries
- `data-offset-pagination` (HIGH) - Prefer Cursor Pagination for Large Lists
- `data-optimistic-concurrency` (CRITICAL) - Guard Read-Modify-Write Updates
- `data-overfetch-select` (HIGH) - Select API Fields Intentionally
- `data-prisma-service-boundary` (HIGH) - Keep ORM Access Behind Feature Services
- `data-race-condition-upsert` (CRITICAL) - Avoid Read-Then-Create Races
- `data-transaction-boundary` (CRITICAL) - Wrap Dependent Writes in One Transaction
- `data-unbounded-result-set` (HIGH) - Bound Every List Query

### Errors & Observability (errors-)

- `errors-domain-exceptions` (HIGH) - Translate Domain Failures to NestJS Exceptions at Boundaries

### Operations & Configuration (ops-)

- `ops-config-validation` (MEDIUM) - Validate Configuration at Startup

### Performance & Runtime (perf-)

- `perf-request-scope` (MEDIUM-HIGH) - Avoid Request-Scoped Providers Unless Required

### Security & Authorization (security-)

- `security-guard-composition` (CRITICAL) - Put Auth and Authorization in Guards

### Testing Strategy (test-)

- `test-provider-overrides` (MEDIUM) - Test NestJS Units with Provider Overrides

### Transports & Integration Patterns (transport-)

- `transport-graphql-resolvers` (MEDIUM) - Keep GraphQL Resolvers Thin
