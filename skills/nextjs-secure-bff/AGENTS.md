> Generated by `bun run build`. Edit `SKILL.md`, `references/`, `dataset/`, or `sources/` instead.

## Skill Description

Use for security-sensitive Next.js backend-for-frontend work involving Route Handlers, Proxy, Server Actions, webhooks, callbacks, data access layers, server-only modules, request validation, rate limits, or public API boundaries. Prefer this skill when a Next.js task exposes HTTP endpoints or moves privileged data across server/client boundaries.

Version: 0.1.0; Framework baseline: v16.2.6.

## Rule Sections

### API Contracts & Validation (api-)

- `api-proxy-headers` (HIGH) - Separate Upstream Request Headers from Client Response Headers
- `api-route-boundaries` (CRITICAL) - Keep Route Handlers at the Public Boundary
- `api-webhook-validation` (CRITICAL) - Validate Webhook and Callback Requests Before Side Effects

### Security & Authorization (security-)

- `security-dal-dto` (CRITICAL) - Isolate Data Access and Return Minimal DTOs
- `security-env-server-only` (CRITICAL) - Keep Secrets Behind Server-only Modules
- `security-server-action-authz` (CRITICAL) - Re-authorize Inside Every Server Action
