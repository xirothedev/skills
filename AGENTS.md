## Rule Sections

### API Contracts & Validation (api-)

- `api-proxy-rename` (MEDIUM) - Use proxy.ts Instead of middleware.ts (Next.js 16+)
- `api-route-handlers` (HIGH) - Write Typed API Route Handlers
- `api-server-actions` (HIGH) - Use Server Actions for Mutations, Not API Routes

### Architecture & Dependency Injection (arch-)

- `arch-app-router` (CRITICAL) - Use App Router as Default Architecture
- `arch-file-conventions` (HIGH) - Follow Next.js File Conventions Correctly

### Bundle Optimization (bundle-)

- `bundle-barrel-imports` (MEDIUM) - Avoid Barrel File Imports
- `bundle-dynamic-imports` (HIGH) - Lazy-Load Heavy Components with next/dynamic
- `bundle-font-optimization` (HIGH) - Optimize Fonts with next/font

### Client Components & Interactivity (client-)

- `client-boundary-placement` (HIGH) - Minimize Client Component Surface Area
- `client-no-forwardref` (MEDIUM) - Use Direct ref Prop (React 19, No forwardRef)
- `client-props-serialization` (MEDIUM) - Pass Serializable Props to Client Components

### Persistence & Transactions (data-)

- `data-fetching-caching` (CRITICAL) - Cache Responses Appropriately with fetch Options
- `data-revalidation` (HIGH) - Revalidate Data with Time-Based or On-Demand Strategies
- `data-unstable-cache` (MEDIUM) - Use unstable_cache for Cross-Request Deduplication

### Operations & Configuration (ops-)

- `ops-config-validation` (MEDIUM) - Validate Next.js Config at Build Time
- `ops-error-boundary` (MEDIUM) - Handle Errors with error.tsx and global-error.tsx

### Performance & Runtime (perf-)

- `perf-image-optimization` (MEDIUM) - Optimize Images with next/image
- `perf-streaming` (HIGH) - Stream UI with Suspense Boundaries

### Prefetch (prefetch-)

- `prefetch-navigation` (MEDIUM-HIGH) - Leverage Automatic Prefetching

### Security & Authorization (security-)

- `security-auth-actions` (HIGH) - Authenticate Server Actions Like API Routes
- `security-env-boundary` (HIGH) - Never Leak Server Environment Variables to Client
- `security-server-validation` (CRITICAL) - Validate Inputs in Server Actions and API Routes

### Server Components & Rendering (server-)

- `server-async-params` (CRITICAL) - Await Route Params in Server Components (Next.js 15+)
- `server-components` (CRITICAL) - Default to Server Components, Opt Into Client When Needed
- `server-no-sg-props` (CRITICAL) - Replace getStaticProps/getStaticPaths with generateStaticParams
- `server-no-ssr-props` (CRITICAL) - Replace getServerSideProps with Server Components

### Testing Strategy (test-)

- `test-component-units` (MEDIUM) - Test Server and Client Components Appropriately
- `test-e2e-flows` (MEDIUM) - Use Playwright for End-to-End Testing
